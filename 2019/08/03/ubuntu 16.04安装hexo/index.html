Ubuntu 16.04 安装 hexo

本文将介绍如何在ubuntu 16.04中搭建hexo静态博客，最终将其部署到Github上。

第一步：安装nodejs

因为hexo是基于nodejs的，所以我们第一步是要进行nodejs的安装。但Ubuntu源中的nodejs是旧版本，所以需要在安装后更新nodejs

sudo apt-get install nodejs
sudo apt install nodejs-legacy
sudo apt install npm

第二步：更换成淘宝的镜像

若不更换，会很慢

sudo npm config set registry https://registry.npm.taobao.org

接着我们可以同行过sudo npm config list来查看是否生效

第三步：安装更新版本的工具

sudo npm install n -g

第四步：更新版本

sudo n stable

如果看到出现installed：版本号则说明安装成功

第五步:安装hexo

sudo npm install -g hexo

注意：若有命令输入后安装不成功，多运行几次命令就可以啦，以及有更新完之后终端输入node -v与npm -v还是以前的版本，这不代表你没有升级哦，只需重启一个新的终端查看版本即可。

第六步：进入我们的hexo

安装成功后，在主目录下建立一个文件夹，我这里就是/home/blog,这个文件夹其实就是我们的博客的所有内容。（所以如果以后出现什么问题，我们不用害怕，大不了直接将相应的文件夹删掉即可）
进入/home/blog文件夹路径下

hexo init

时间会有些长，结束之后我们进入/home/blog后可以看到：

然后我们进行hexo的生成以及开启

hexo g
hexo s

启动成功之后我们进入本地localhost:4000就可以看到自己的blog啦！

部署hexo到Github

作为博客我们当然不可以只在本地打开，所以我们将其部署到Github中，更加方便浏览已经更加专业。

第一步:创建一个新的仓库

登录到Github上。然后New Repository，注意用户部署个人博客的github仓库的命名必须符合特定的要求才可以（一定要与你的用户名相同）例如：

创建之后呢你会看到一个SSH(类似于网址)

第二步：在/home/blog目录下装一个部署插件

sudo npm install –save hexo-deployer-git

注意：若一次不成功，多试几次就好了

第三步：进行_config.yml的设置

打开_config.yml，在#Deployment下进行设置


注意：上图的repo：的链接就第一步中新建仓库后生成的SSH链接

第四步：进行部署

在修改完_config.yml之后我们就可以进项部署了

hexo d

之后终端会提醒你输入Github的用户名以及密码。

第五步：启动hexo

在url中输入：https://t yourName.github.io，多刷新几次就会出现啦！

附录：

在部署的过程中可能会报错，意思是提醒你没有配置你的github，我们可直接在终端中输入：

git config –global user.name “yourName”
git config –global user.eamil “email@example.com

其中yourname 是输入你自己的用户名，email@example.com输入你自己的注册邮箱。
